## D07 - Основы оркестрации контейнеров


Использование инструментов Docker Compose и Docker Swarm для совместного запуска контейнеров и их простейшей оркестрации.

### О проекте

Проект посвящён изучению оркестрации контейнеров. На первом этапе микросервисное приложение было упаковано в Docker-контейнеры и запущено с помощью Docker Compose. На втором этапе с использованием Vagrant был развёрнут кластер Docker Swarm из трёх узлов, на котором запущен стек сервисов с настройкой проксирования через nginx и визуализацией через Portainer.

Работа выполнялась на виртуальных машинах с **Ubuntu 22.04.5 LTS**.

### Ключевые навыки

В ходе выполнения проекта были освоены и применены следующие технологии и инструменты:

- **Docker:** Написание Dockerfile для микросервисов, сборка и оптимизация образов
- **Docker Compose:** Описание многоконтейнерного приложения, проброс портов, сетевые настройки
- **Docker Swarm:** Инициализация кластера, управление узлами (manager/worker), развёртывание стека сервисов
- **Оверлейная сеть:** Настройка сети для взаимодействия сервисов в кластере
- **Nginx:** Настройка прокси-сервера для доступа к внутренним сервисам
- **Vagrant:** Создание и управление виртуальными машинами через Vagrantfile
- **Portainer:** Установка и использование веб-интерфейса для визуализации кластера
- **Docker Hub:** Загрузка образов в публичный реестр
- **Postman:** Тестирование API микросервисного приложения

### Выполненные задачи

Проект состоит из трёх частей, каждая из которых строится на предыдущей:

1.  **Запуск с Docker Compose:**
    - Написание Dockerfile для каждого микросервиса из папки `src`.
    - Создание `docker-compose.yml` для корректного взаимодействия сервисов.
    - Сборка и локальный запуск веб-сервиса с пробросом портов для gateway и session service.
    - Успешный прогон заготовленных тестов через Postman.
2.  **Создание виртуальных машин:**
    - Установка и инициализация Vagrant.
    - Написание Vagrantfile для одной виртуальной машины.
    - Перенос исходного кода веб-сервиса в рабочую директорию виртуальной машины.
    - Проверка корректности переноса, остановка и уничтожение машины.
3.  **Создание Docker Swarm:**
    - Модификация Vagrantfile для создания трёх машин: manager01, worker01, worker02.
    - Написание shell-скриптов для установки Docker, инициализации и подключения к Swarm.
    - Загрузка собранных образов на Docker Hub и обновление Docker Compose файла.
    - Запуск стека сервисов в кластере.
    - Настройка прокси на базе nginx для доступа к gateway и session service по оверлейной сети.
    - Прогон тестов через Postman и проверка успешного прохождения.
    - Отображение распределения контейнеров по узлам с помощью команд Docker.
    - Установка Portainer отдельным стеком и визуализация распределения задач по узлам.
